# 05 文档体系说明

LoopPilot 文档分三层，降低认知负担：**中文理解，英文执行，混合提示词**。

## 三层架构

```text
┌─────────────────────────────────────────────────────────┐
│ Layer 1  中文认知层（人读）                                │
│   README.md · docs/zh/ · docs/development/README.md     │
│   prompts/ 中文目标 · logs/ 中文决策记录                   │
├─────────────────────────────────────────────────────────┤
│ Layer 2  英文执行层（AI / 代码 / CI）                    │
│   src/loop_pilot/ · config/*.yaml · CLI · schema        │
│   docs/en-core.md · docs/development/00–39 编号规格      │
├─────────────────────────────────────────────────────────┤
│ Layer 3  混合提示词层                                     │
│   prompts/*.md — 中文：目标/做什么                         │
│                 English: Task, Constraints, Acceptance    │
└─────────────────────────────────────────────────────────┘
```

## 各层职责

### Layer 1 — 中文认知层

**给谁：** 项目所有者、日常维护者  
**解决：** 30 秒内知道「这是什么、现在到哪了、怎么跑」  
**入口：** [README.md](../../README.md) → [docs/zh/README.md](README.md)

不追求覆盖所有细节；深入时跳转到 Layer 2 编号文档。

### Layer 2 — 英文执行层

**给谁：** Cursor Agent、CI、未来贡献者写代码/测试  
**解决：** 接口契约、状态机、Adapter 协议、验收文件名  
**入口：** [docs/en-core.md](../en-core.md) → [docs/development/README.md](../development/README.md)

编号文档（`00-system-definition.md` … `39-next-steps-0.3.md`）**保持英文文件名**，内容为权威规格，**不整篇翻译**。

### Layer 3 — 混合提示词层

**给谁：** 把任务交给 AI 实施时复制粘贴  
**结构：**

- 中文：目标、背景、当前版本、不要做什么
- English：Task scope, Constraints, Acceptance criteria, Read-before-coding list

**目录：** [prompts/](../../prompts/)

## 链接约定

| 从 | 到 | 说明 |
|----|-----|------|
| README | `docs/zh/` | 日常阅读 |
| README | `docs/en-core.md` | 一行英文规格入口 |
| `docs/zh/*` | `docs/development/*` | 需要细节时 |
| `docs/en-core.md` | `docs/zh/README.md` | 反向链接 |
| prompts | 两者 | 实施前读英文编号 doc |

## 不做什么

- 不做：删除现有 `docs/development/*.md`
- 不做：全文双语同步（维护成本过高）
- 不做：改源码/CLI/配置 schema 的语言
- 不做：Phase 2 的 `docs/getting-started/` 完整树（尚未建设）

## 维护规则（权威）

放置位置、何时写中文/英文、验收双语策略、链接约定 — 见 **[10-中英文文档管理.md](10-中英文文档管理.md)**（2026-06-21 起为维护权威）。
