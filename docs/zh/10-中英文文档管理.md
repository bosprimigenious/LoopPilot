# 10 中英文文档管理

本文是 LoopPilot **文档放置与维护的权威说明**。三层架构概览仍见 [05-文档体系说明.md](05-文档体系说明.md)；**维护规则、验收双语策略、链接约定以本文为准**。

决策记录：[logs/2026-06-21-zh-en-doc-management.md](../development/logs/2026-06-21-zh-en-doc-management.md)

## 1. 核心原则

| 语言 | 角色 | 一句话 |
|------|------|--------|
| **中文** | 认知层 | 理解、规划、日常跑命令 |
| **英文** | 执行层 | AI 实施、CI、接口契约、验收文件名 |
| **混合** | 提示词层 | 中文目标 + English constraints |

- **不追求全文双语同步** — 维护成本过高，且英文编号文档已是交叉引用枢纽
- **不删除**已有 `docs/development/*.md`
- **文档中不使用 emoji**
- **不改**源码、CLI 子命令名、config 键名、schema 字段的语言

## 2. 放哪里

```text
Layer 1  中文认知层
  README.md                    # 项目主入口（中文 + 一行英文规格链接）
  docs/zh/00–10*.md            #  numbered 中文认知文档
  docs/development/logs/*.md   # 决策/验收跑记录（中文为主，可混英文命令）
  prompts/*.md                 # 中文目标段 + English Task/Constraints

Layer 2  英文执行层
  docs/en-core.md              # 最小英文接口规格（AI/CI 入口）
  docs/development/00–42*.md   #  numbered 权威规格（保持英文文件名）
  src/ loop_pilot/ config/     # 代码与配置即规格

Layer 3  混合
  prompts/CURSOR_*.md          # 复制给 Agent 的实施任务
```

## 3. 何时写中文 / 何时写英文

| 场景 | 写哪里 | 语言 |
|------|--------|------|
| 版本路线图、阶段边界、个人优先战略 | `docs/zh/03`、`08`、`09` + `development/34`、`42` | 中文理解 + 英文权威规格 |
| 验收清单（DoD 勾选、命令、证据） | `development/32、35、36…` | **英文文件名**；中文摘要放 `docs/zh/07` 等 |
| 验收跑记录、 pivot 决策 | `development/logs/` | 中文为主 |
| Adapter/状态机/Schema 细节 | `development/19、18、24…` | 英文 |
| 给 Cursor 的实施任务 | `prompts/` | 混合 |
| 新人 30 秒了解项目 | `README.md` → `docs/zh/00` | 中文 |

**规则**：新能力先改英文编号 spec（或 logs 决策），再补/更新对应 `docs/zh/` 认知页；**不要求**英文 spec 逐段翻译。

## 4. 谁维护

| 维护者 | 职责 |
|--------|------|
| 项目所有者 | `docs/zh/` 路线图与验收理解、logs 决策、`README.md` |
| AI Agent / 实施 PR | `development/` 编号 spec、验收清单勾选、`en-core.md` 最小更新 |
| 两者 | `development/README.md` 索引与 **zh ↔ en 映射表** |

合并 PR 时：若改阶段边界，须同时更新 `docs/zh/03`（或 07–09）与 `development/34`（或 42），并在 logs 留一句决策（可选）。

## 5. 链接规则

| 从 | 到 | 写法 |
|----|-----|------|
| `docs/zh/*` | `docs/development/*` | 相对路径 `../development/36-…` |
| `docs/development/README.md` | `docs/zh/*` | `../zh/07-…` |
| `README.md` | 中文 + 英文 | 中文正文 + `English spec: docs/en-core.md` |
| `docs/en-core.md` | 中文 | `docs/zh/README.md` |
| 验收中文页 | 英文清单 | 文首「英文权威清单见 …」 |

**禁止**：只写文件名不写相对路径；在中文页复制整篇英文验收正文。

## 7. 编号约定

| 目录 | 编号 | 说明 |
|------|------|------|
| `docs/zh/` | `00`–`10` | 中文认知；新增主题递增，不插入 development 编号 |
| `docs/development/` | `00`–`42` | 英文规格；**不重命名、不移动**已有文件 |
| `logs/` | 日期前缀 | `YYYY-MM-DD-主题.md` |

## 8. 不做什么

- 不做 `docs/development/` 整篇机器翻译
- 不做 `docs/getting-started/` 完整双语树（尚未建设；1.0 再评估）
- 不在中文页重复 Adapter 五方法、状态机全表（链到 `en-core` / `development/18`）
- 不用 emoji 标记状态（用「已完成 / 进行中 / 规划」文字）
