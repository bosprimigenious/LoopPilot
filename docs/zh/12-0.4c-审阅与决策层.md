# 12 0.4-c 审阅与决策层

本文说明 **0.4-c 个人审阅** 的目标与 CLI。英文权威规格见 [development/40-personal-daily-loop-0.4-spec.md](../development/40-personal-daily-loop-0.4-spec.md) §3.3；验收清单见 [45-personal-daily-loop-0.4c-acceptance.md](../development/45-personal-daily-loop-0.4c-acceptance.md)（待交付）。

## 定位

0.4-c 让「跑完 Loop」之后有一条可执行的审阅链：

```text
run 结束 → review list → 人读 review_required.md → approve / reject / defer / cancel
```

前置：**0.4-a** SQLite + recovery、**0.4-b** inbox/queue/today 已完成或并行。

## 核心 CLI

| 命令 | 作用 |
|------|------|
| `loop-pilot review list` | 待审 run 列表 |
| `loop-pilot approve <run-id>` | 接受结果 |
| `loop-pilot reject <run-id> --reason "..."` | 拒绝并记录原因 |
| `loop-pilot defer <run-id> [--until DATE]` | 推迟 |
| `loop-pilot cancel <run-id>` | 用户取消 |
| `loop-pilot resume <run-id>` | 从合法 checkpoint 恢复 |

## 输出接口（必读）

每次进入审阅队列的 run **必须** 具备：

- **`review_required.md`** — 人读入口（推荐动作、理由、清单）
- **`gate_result.json`** — 机器门禁结果（`pass` / `needs_review` / `blocked`）

完整文件布局与矩阵见 **[13-输出接口-人看MD机器看JSON.md](13-输出接口-人看MD机器看JSON.md)** 与英文 [47-output-interface-spec.md](../development/47-output-interface-spec.md)。

## 与 agentic-rubric-runner

可选外部评测将结果写入 `gate_result.json`；人仍通过 `review_required.md` 做最终决定。详见 [13-输出接口-人看MD机器看JSON.md](13-输出接口-人看MD机器看JSON.md) §与 agentic-rubric-runner 的关系。
