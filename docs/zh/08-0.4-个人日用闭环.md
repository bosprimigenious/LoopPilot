# 08 0.4 个人日用闭环

本文说明 **0.4 Personal Recovery & Daily Loop** 的目标、范围与四子阶段。英文权威规格见 [development/40-personal-daily-loop-0.4-spec.md](../development/40-personal-daily-loop-0.4-spec.md)。

## 定位：0.3 与 0.4 的区别

| | **0.3 Adapter MVP** | **0.4 Personal Daily Loop** |
|---|---|---|
| 核心问题 | 真实 Adapter 能否安全跑通？ | 我能否**每天**靠它管任务、审结果、看总结？ |
| 状态存储 | JSON 可接受 | **SQLite 生产路径** |
| 任务 OS | 无专用 CLI | **inbox / queue / today** |
| 审阅 | Markdown 侧车 | **review list + approve/reject/defer/cancel** |
| 总结 | 单次 run 报告 | **summary today/week + daily-summary.md** |
| 调度 | 无 | **schedule print / install --dry-run** |
| 触发方式 | 开发者手动 `run` | 个人日用链 + 可选 OS 调度预览 |

**旧称「V1」的主体是 0.4，不是 0.3。** 早期文档里把 Adapter 与 recovery/调度混叫 V1，现拆分为 0.3 + 0.4。

## 0.4 要达成什么

LoopPilot 成为**个人每日任务 OS**：

1. **状态可靠** — SQLite 可备份、可迁移、中断可恢复
2. **任务入口清晰** — Anything → Inbox → Queue → Today
3. **审阅可执行** — 一条命令看清待处理，四种审阅动作可审计
4. **总结可复盘** — 今日/本周摘要；调度仅 dry-run，不悄悄写系统任务

成功定义（维护者自用）：**连续 5 个工作日**仅用 CLI 完成「收任务 → 跑 Loop → 审结果 → 看总结」，不手工改 SQLite。

## 验收摘要（10 项，0.4 全阶段）

1. SQLite 事务：中断无半写终态；migration 可重复
2. ACTING 中断 → 默认 BLOCKED；`recovery-scan` 可见
3. `resume` 合法 checkpoint 可恢复，不重复 artifact
4. inbox → queue → today 链路可审计
5. approve/reject/defer/cancel 写入 SQLite，trace 可审计
6. `run all` 每日链：单 Loop 失败不抹掉其他结果
7. `summary today` 产出 daily-summary.md
8. `schedule print` / `install --dry-run` 不写系统任务
9. `state_backend=json` 时 0.1 基线不退化
10. 锁与 stale run 可识别

详细命令与 schema 见英文 [40](../development/40-personal-daily-loop-0.4-spec.md)。
