# 02 三条循环

## 关系图

```text
DailyNewsLoop ──候选任务──┬──> InternLoop（intern-inbox.md）
                          └──> PaperLoop（paper-inbox.md）

InternLoop  = 昨日问题 → 修复 → 测试 → 诊断 → 报告
PaperLoop   = 论文缺口 → 查证 → 修订 → 证据检查 → 报告
DailyNews   = 离线快照 → 去重筛选 → 日报 + Inbox 候选
```

DailyNews **不能直接改代码或论文**，只能通过 Inbox 传递候选。

## InternLoop（主 Loop）

**目标：** 一个经过测试的代码修复，或明确的阻塞诊断。

**典型路径：** 观察仓库 → 选一个任务 → 计划 → Policy 检查 → 改代码 → 跑测试 → 评估 → 报告。

**完成标准：** 验收条件满足、强制测试通过、无越界 diff、有开发报告。仅生成补丁未验证 **不算完成**。

**规格：** [development/03-intern-loop.md](../development/03-intern-loop.md)

## PaperLoop（主 Loop）

**目标：** 有证据的论文推进，或明确的研究方向。

**典型路径：** 观察论文副本 → 选缺口 → 查证/修订 → 引用与证据检查 → 报告。

**完成标准：** 论点可核验、引用真实、实验数字一致。仅加文字 **不算完成**。

**规格：** [development/04-paper-loop.md](../development/04-paper-loop.md)

## DailyNewsLoop（小 Loop）

**目标：** 少量高价值工程/研究/时政财经信号。

**典型路径：** 读离线快照 → 去重、分级 → 写日报 → 路由候选到两个 Inbox。

**完成标准：** 增量抓取、来源分级、条数在配置上限内。堆链接 **不算完成**。

**规格：** [development/05-daily-news-loop.md](../development/05-daily-news-loop.md)

## 共同约束

- 默认预算：**约 30 分钟**，最多 3 轮
- 第 25 分钟后不再开新大动作，留 5 分钟收尾写报告
- 每次 Run 有唯一 `run_id`，状态机驱动，非法跳转不允许

**运行机制：** [development/02-runtime-mechanism.md](../development/02-runtime-mechanism.md)
