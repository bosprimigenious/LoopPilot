# 15 0.5 安全自治

本文说明 **0.5 安全自治（Safe Autonomy）** 的目标、顺序与验收。英文权威规格见 [development/50-personal-daily-loop-0.5-spec.md](../development/50-personal-daily-loop-0.5-spec.md)；决策理由见 [52-0.5-revised-plan-rationale.md](../development/52-0.5-revised-plan-rationale.md)；验收清单见 [53-0.5-acceptance.md](../development/53-0.5-acceptance.md)。

## 一句话定义

**0.5 = 在真实系统定时任务下，安全、无人值守地跑每日闭环** — 先过 SafetyGate，再装调度，再 `run daily --unattended --safe`；**不需要后台 daemon**；**绝不**自动 approve。

## 与 0.4-d 的区别

| 能力 | 0.4-d | 0.5 |
|------|-------|-----|
| `schedule install --dry-run` | 只写预览文件 | 保持不变 |
| `schedule install --yes` | **拒绝** | 经 SafetyGate 后**真装**系统调度 |
| `run daily --dry-run` | 安全编排预览 | 保持不变 |
| `run daily --unattended --safe` | 不存在 | **核心交付** |
| 后台 daemon | 无 | **仍然无**（OS 调度器直接调 CLI） |
| 真实 Adapter 无人值守 | 禁止 | 禁止（safe 配置下） |

```text
Step 3  run daily --unattended --safe  ← 调度器调用，无 daemon
Step 4  崩溃恢复与可观测性
```

## 四个子里程碑

| 子阶段 | 内容 |
|--------|------|
| **0.5-a** | SafetyGate + 真实 schedule install |
| **0.5-b** | 无人值守 safe 每日运行 |
| **0.5-c** | 崩溃/恢复/日志可观测 |
| **0.5-d** | 连续 5 个工作日真人试用 |

**0.5-prep（可与 0.4-c 并行）：** SafetyGate 类型、调度 dry-run 打磨、status/审计只读扩展、日志 schema 设计。**不允许**真装调度或真无人值守。

## SafetyGate 是什么

所有「危险动作」在执行前必须经过统一门禁：

- 装系统定时任务
- 非 dry-run 的每日运行
- （未来）更高自主级别

门禁结果写入 **JSON 审计记录**（不是让人去读 Markdown 判状态）。与 0.4-c 人工审阅、0.3 ToolBroker **叠加**，不互相替代。

## 为什么没有 daemon

0.5 采用最常见个人运维模型：

```text
cron / systemd timer / Windows 计划任务
        ↓
loop-pilot run daily --unattended --safe
```

不另起常驻进程：少一层监督、日志在系统调度器里、责任边界清晰。

## 安全性（红线）

| 行为 | 0.5 |
|------|-----|
| 自动 approve / commit / push | **禁止** |
| safe 模式下真实 Adapter | **禁止** |
| 跳过 0.4-c 宣布 0.5 READY | **禁止** |
| 无 SafetyGate 装调度 | **禁止** |

## 核心 CLI（实现后）

```bash
# 0.4-d 回归（须保持可用）
loop-pilot schedule install --dry-run
loop-pilot run daily --dry-run

# 0.5 新增
loop-pilot schedule install --yes --confirm-schedule   # 需配置 allow_install
loop-pilot run daily --unattended --safe
```

需 `state_backend=sqlite`（与 0.4-a 相同）。

## 配置要点（示意）

```yaml
schedule:
  allow_install: false    # 默认关；验收时手动开

safety:
  safe_profile:
    allow_real_adapters: false
    allow_loop_execution: none
```
