# 09 1.x 个人到协作

本文说明 **1.0 → 1.3** 四段式路线：先个人稳定，再个人智能，再受控协作，最后团队/云 preview。**Team 与 Cloud 不进 0.x**，完整英文规格见 [development/42-1x-roadmap-personal-to-collaboration.md](../development/42-1x-roadmap-personal-to-collaboration.md)。

## 为什么 Team 后移

2026-06-21 战略 pivot：LoopPilot 首先是**一个人每天真的能用**的开发助手 + 论文推进器 + 信息筛选器 + 每日任务 OS。旧规划把 team-cloud 放在 0.8，现拆到 **1.2（受控协作）** 与 **1.3（团队 preview）**。

```text
0.9 RC ──► 1.0 个人稳定 ──► 1.1 个人智能 ──► 1.2 受控协作 ──► 1.3 团队 preview
```

**禁止**：1.0 后立刻做大团队版或完整 SaaS。

## 总览

| 版本 | 名称 | 核心 | 状态 |
|------|------|------|------|
| **1.0** | Personal Stable | 单人长期 semver 承诺；30 天日用 | 规划 |
| **1.1** | Personal Intelligence | 记忆、规划、优先级、主动建议 | 规划 |
| **1.2** | Controlled Collaboration | 文件包 export/import、handoff | 规划 |
| **1.3** | Team / Cloud Preview | RBAC、多项目、Dashboard preview | 后置规划 |

## 1.0 Personal Stable

**第一个稳定版本** — 不追团队化，证明个人可**连续 30 天**日用。

必备能力（在 0.4 日用链 GA 基础上）：

- 稳定 CLI：inbox / queue / today / review / summary / recovery / db
- 稳定 SQLite migration（additive-only）
- 稳定 Adapter 默认 gate 与 ToolBroker
- 稳定 `summary today/week`、备份与恢复
- PyPI `1.0.0` 与文档稳定承诺（承接 0.5/0.8 个人 beta 口径）

**不含**：多用户 RBAC、共享审批队列、云端 sync、完整 Dashboard。

## 1.1 Personal Intelligence

承接 **0.6 Personal Memory & Evaluation**：

- 跨日记忆：昨日 next action、审批决定、Inbox 遗留
- 跨 Loop 规划与优先级解释
- 周报与效率指标（本地，无公开排行榜）

**仍是单人** — 不是团队功能。

## 1.2 Controlled Collaboration

**受控协作** — 少量可信协作者，**无云端账号体系**：

- 报告/任务/审阅状态的**文件包** export/import
- handoff：把「待你审」的包交给协作者，审完再 import
- 协作者 profile（本地配置，非 IdP）

旧 33 文档 §0.8 里偏「文件交换」的能力落在这里；**不含** RBAC 与在线 Dashboard。

## 1.3 Team / Cloud Preview

**preview only** — 旧 **0.8 team-cloud** 的主要能力迁此（[33 §0.8](../development/33-version-roadmap.md) 标注 legacy）：

| 能力 | 新归属 |
|------|--------|
| 多项目 workspace | 1.3 |
| RBAC / 共享审批 | 1.3 |
| 本地 Dashboard :7860 | 1.3 preview |
| 团队插件策略 | 1.3 |
| Cloud SaaS / 多租户计费 | **永不**（local-first 原则） |

1.3 是**预览**，不是 GA 团队版；完整 SaaS 若做，也在更后 major（如 2.0）且需单独产品决策。

| 旧称呼 | 现版本 |
|--------|--------|

| 0.8 team-cloud（33 文档） | **1.2 + 1.3**（移出 0.x） |
| 笼统「1.1+ Team」 | **1.1–1.3 四段式** |
| V1 SQLite/recovery | **0.4**（不是 1.0 才做 recovery） |

## 核心原则（1.x）

1. **1.0 之前**不承诺稳定接口
2. **1.0 之后** 1.x 仅 additive；breaking → 2.0
3. **团队协作不应早于个人稳定** — 不建议跳过 1.0/1.1
4. **云端不应早于本地安全模型**
5. 所有自动化须可审计、可暂停、可恢复
