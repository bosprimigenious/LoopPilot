# 14 0.4-d 日汇总与调度预览

本文说明 **0.4-d 每日总结与调度预览** 的目标与验收。英文权威规格见 [development/48-personal-daily-loop-0.4d-acceptance.md](../development/48-personal-daily-loop-0.4d-acceptance.md)；架构见 [49-daily-summary-engine-design.md](../development/49-daily-summary-engine-design.md)。

## 一句话定义

**0.4-d = 系统是否具备「可日常使用 + 可调度预览 + 可复盘」完整闭环** — 不是功能演示验收，而是「能不能当每日仪表盘用」。

## 与 0.4-a/b/c 的区别

| 子阶段 | 焦点 | 0.4-d 不负责 |
|--------|------|--------------|
| **0.4-a** | SQLite、db CLI、recovery-scan | — |
| **0.4-b** | inbox / queue / today 任务入口 | — |
| **0.4-c** | review list、approve/reject/defer | — |
| **0.4-d** | summary today/week、schedule 预览、run daily dry-run | 真实调度安装、真实 Loop 执行、自动 approve |

0.4-d 建立在 a/b/c 之上：**汇总层读真实 SQLite 与 artifacts**，不是 mock 数据。

## 四层验收

### 第 1 层 — Summary（人读仪表盘）

- `loop-pilot summary today` → `var/artifacts/daily/<日期>/daily-summary.md`
- `loop-pilot summary week` → `var/artifacts/weekly/<周>/week-summary.md`
- daily-summary 七段：结论、Today、Runs、Needs Review、Inbox、Blocked、Tomorrow

### 第 2 层 — 数据聚合（SQLite + 产物）

- 从 `inbox_items`、`queue_items`、`today_items`、`runs`、`review_items`、`task_events` 聚合
- 计数可与 SQL 查询对账
- 每条 run 可追溯到 `run_id` 与 artifact 路径

### 第 3 层 — 调度预览（只看不装）

- `schedule print` — 打印 cron/systemd/计划任务配置文本
- `schedule install --dry-run` — 写预览文件，**不写系统**
- `schedule install --yes` — 0.4-d 必须拒绝或推迟到 0.5

### 第 4 层 — 每日 dry-run（安全编排）

- `loop-pilot run daily --dry-run` — verify + recovery-scan + today 预览 + 刷新 summary
- **不**执行真实 intern/paper/daily-news
- **不**调用 Adapter / 外部 API

## 闭环条件

完整个人日用链：

```text
Inbox → Queue → Today → Run → Review → Summary
```

验收时：inbox 有项、queue 已 promote、today 有焦点、run 有记录（可 dry-run）、review 有待办时，`summary today` 对应段落均应有内容或明确「无」占位。

## 安全性

| 命令 | 0.4-d 行为 |
|------|------------|
| `schedule print` | 只 stdout，不改 OS |
| `schedule install --dry-run` | 只写 `var/artifacts/schedule/schedule-preview.md` |
| `run daily --dry-run` | 只编排预览，不跑 Loop |
| 自动 approve / commit / push | **禁止** |

## 核心 CLI

```bash
loop-pilot summary today
loop-pilot summary week
loop-pilot schedule print
loop-pilot schedule install --dry-run
loop-pilot run daily --dry-run
python scripts/verify_0_4d_acceptance.py
```

需 `state_backend=sqlite`（与 0.4-a 相同验收配置）。
