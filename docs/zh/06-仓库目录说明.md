# 06 仓库目录说明

本文说明 LoopPilot 仓库的**顶层目录分工**。目标是：一眼知道「代码在哪、文档在哪、本地运行产物在哪」，避免误删或误提交。

## 顶层一览

```text
LoopPilot/
├── README.md                 # 项目主入口（中文认知 + 英文规格链接）
├── DEVELOPMENT_PLAN.md       # 开发计划与里程碑
├── DELIVERY.md               # 交付说明
├── CHANGELOG.md              # 版本变更记录
├── ideas.md                  # 早期想法与追溯（保留，不删）
├── pyproject.toml            # Python 包与 dev 依赖
├── config/                   # 默认 YAML 配置（可版本化）
├── src/loop_pilot/           # 主 Python 包（CLI、Runtime、Loops、Adapters）
├── tests/                    # 单元 / 集成 / 场景测试与 fixtures
├── examples/                 # 可运行的 demo workspace
├── schemas/                  # JSON Schema（如 artifact-manifest）
├── templates/                # Markdown 报告与审批模板
├── scripts/                  # bootstrap、回归、备份等运维脚本
├── prompts/                  # 给 AI 的双语实施提示词
├── docs/                     # 文档根（见下文）
└── var/                      # 本地运行时目录（内容不提交，见 .gitignore）
```

## 代码与配置

| 目录 | 用途 | 是否提交 |
|------|------|----------|
| `src/loop_pilot/` | 产品代码：CLI、编排、三条 Loop、Adapter、存储 | 是 |
| `config/` | `loop-pilot.yaml` 等默认配置 | 是 |
| `tests/` | pytest 测试与 golden fixtures | 是 |
| `examples/` | Intern / Paper / DailyNews demo workspace | 是 |
| `schemas/` | 产物与清单的结构定义 | 是 |
| `templates/` | 报告渲染模板 | 是 |
| `scripts/` | 本地初始化、迁移、备份、回归 | 是 |

**不要**随意移动 `src/loop_pilot/` 包结构或改 import 路径——CLI 入口、测试、文档多处引用现有布局。

## 本地运行时 `var/`

`var/` 存放**本机**状态与产物，**不进入版本控制**（见 `.gitignore`）。典型结构：

```text
var/
├── state/          # SQLite、run 记录、manifest
├── artifacts/      # 每次 run 的报告、trace、patch 等
├── checkpoints/    # 检查点
├── locks/          # 进程锁（运行时创建）
└── logs/           # 可选本地日志
```

首次可用 `python scripts/bootstrap_local.py` 创建 `var/state`、`var/artifacts`、`var/checkpoints`。

仓库内仅保留 `var/.gitkeep` 占位；**不要**提交 `var/artifacts/` 下的 run 输出。

## 应忽略的生成物

以下仅存在于本地，已在 `.gitignore` 中排除，清理时可直接删除：

- `__pycache__/`、`*.pyc`
- `.pytest_cache/`、`.mypy_cache/`、`.ruff_cache/`
- `*.egg-info/`、`dist/`、`build/`
- `var/` 下除 `.gitkeep` 外的全部内容

## 本次未改动的部分（有意保留）

以下目录/命名**未做重构**，避免破坏现有引用与 CI：

- `docs/development/` 编号文件名（`00`–`39`）— 大量交叉链接
- `src/loop_pilot/` 子包划分 — 与架构文档一致
- `config/*.yaml` 键名与路径 — 与 `config.py` 绑定
- `tests/` 与 `tests/fixtures/` 布局 — 与回归脚本一致
- `ideas.md`、`DELIVERY.md` — 历史与交付追溯

## Phase 3 可选（仅文档建议，未实施）

1. 将 `docs/development/` 按主题分子目录（如 `architecture/`、`acceptance/`），并保留编号别名或索引表。
2. 根目录增加 `CONTRIBUTING.md`，指向 `docs/zh/` 与 `docs/development/`。
3. 统一 `examples/` 与 `tests/fixtures/` 的 README 模板（内容仍分属测试与 demo）。

当前策略：**先文档化布局，再按需小步移动**。
