# 01 系统是什么

## 一句话

LoopPilot 是**只服务你本人**的本地 AI 工作闭环：在有限时间和预算内，真正关闭一个问题，或准确指出下一步该推进什么。

## 核心性质

| 性质 | 含义 |
|------|------|
| **真实** | 读真实仓库、论文、测试和来源（非编造） |
| **可控** | 路径、命令、时间、轮次、费用、审批都有硬约束 |
| **闭环** | 验证失败 → 诊断 → 有限重试，不是一次性生成 |
| **可审计** | 每个结论可追溯到日志、diff、实验或外部来源 |
| **能停止** | 证据不足、范围过大、风险过高时安全终止 |

## 架构分层（简化）

```text
CLI / 调度
    ↓
控制面（Orchestrator、状态机、预算、Policy、锁）
    ↓
三条 Loop 定义（Intern / Paper / DailyNews）
    ↓
能力面（Agent、Skill、Tool、Adapter、ModelRouter）
    ↓
状态与产物（JSON/JSONL → 未来 SQLite；Markdown 报告）
```

**关键分离：** 模型在能力面，**不能直接改状态、延预算或跳过 Policy Gate**。

## 用户可见产出

全部是 **Markdown**：

- 开发报告、论文修订报告、DailyNews 日报
- 诊断、阻塞说明、审批请求（0.4 前多为 `review-required.md`）
- 每日总览

机器侧用 JSON 快照、JSONL 事件、YAML 配置；它们不替代 Markdown 结果。

## 明确不做

- 不自动 push、merge、部署、投稿
- 不编造引用、数字、新闻
- 不做团队/多租户/商业功能
- 不把「Agent 数量」当能力指标
