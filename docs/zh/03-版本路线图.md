# 03 版本路线图

LoopPilot 使用 **0.x 个人优先版本体系**（不是简单的 V1/V2/V3 对外称呼）。

> **战略口径（2026-06-21）**：LoopPilot = **个人开发助手 + 论文推进器 + 信息筛选器 + 每日任务 OS**。0.x 目标是「**一个人每天真的能用**」；Team / Cloud / Public onboarding 放在 **1.1+**（详见 [09-1.x-个人到协作.md](09-1.x-个人到协作.md)）。

## 总览

| 版本 | 名称 | 核心 | 状态 |
|------|------|------|------|
| **0.1** | Mini-MVP | 三条 fixture dry-run，MockAdapter | 已完成 |
| **0.2** | Practical MVP | 受控 demo workspace，报告链路 | 已验收 `v0.2.0a1` |

| **0.4** | Personal Recovery & Daily Loop | SQLite、inbox/queue/today、review、summary | 下一焦点 |
| **0.5** | Personal Beta | init personal、稳定日用、可选 PyPI | 规划 |
| **0.6** | Personal Memory & Evaluation | 长期记忆、周报、效率指标 | 规划 |
| **0.7** | Personal Extensions | 本地 connector/skill | 规划 |
| **0.8** | Optional Public Beta | 公开文档/示例（仍以个人为核心） | 规划 |

## 各版本一句话

**0.1** — 证明编排、状态机、Policy Gate、产物清单在离线 fixture 下能跑通。

**0.2** — 在 `examples/*_demo` 等受控 workspace 上验证业务逻辑；仍默认 Mock + dry-run。

**0.3** — 接入 Cursor CLI、OpenAI-compatible 等真实 Adapter，但 **默认关闭**（`allow_real_adapters=false`）；ToolBroker 管控文件/git/命令。详见 [07-0.3-Adapter验收说明.md](07-0.3-Adapter验收说明.md)。

**0.4** — **个人每日任务 OS**：状态可靠 → 任务入口 → 审阅 → 总结与调度 dry-run。详见 [08-0.4-个人日用闭环.md](08-0.4-个人日用闭环.md)。

**0.5** — **个人 Beta**：换机可装、配置可迁移；PyPI **可选**，不再急 public onboarding。

**0.6–0.7** — 个人记忆与评测、本地扩展（**非**公共插件市场）。

**0.8** — 可选公开发布与文档；仍以个人用户为主。

**1.0** — 个人长期稳定 semver 承诺。

**1.1–1.3** — 个人智能 → 受控协作 → 团队/云 preview。详见 [09-1.x-个人到协作.md](09-1.x-个人到协作.md)。

## 0.4 四子阶段

| 子阶段 | 内容 |
|--------|------|
| **0.4-a** | 状态可靠：`db status/migrate/backup/verify`，`recovery-scan` |
| **0.4-b** | 任务入口：`inbox`、`queue`、`today` |
| **0.4-c** | 个人审阅：`review list`，`approve`/`reject`/`defer`/`cancel` |
| **0.4-d** | 每日总结：`summary today/week`，`schedule print/dry-run` |

**0.4 不做**：团队、Web Dashboard、云同步、插件市场、PyPI onboarding。
