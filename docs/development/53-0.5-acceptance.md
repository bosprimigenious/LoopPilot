## One-line definition

**0.5** validates whether LoopPilot can **safely run on a real OS schedule** — SafetyGate before side effects, `schedule install --yes` audited, `run daily --unattended --safe` without daemon, crash/recovery observable — while **preserving human review** and **never** auto-approving.

## 0.5-a — SafetyGate + Schedule Install

### Scope

| In scope | Out of scope |
|----------|--------------|
| `src/loop_pilot/safety/` v1 | Unattended daily execution |
| `schedule install --yes` through gate | Daemon / supervisor |
| Gate audit records | Real adapter unattended |
| `schedule install --dry-run` unchanged | Auto-approve |

### Checklist

| # | Item | Pass criteria |
|---|------|---------------|
| A1 | SafetyGate module exists | `src/loop_pilot/safety/gate.py`, `policy.py`, `audit.py` importable |
| A2 | Deny by default | `schedule install --yes` without `schedule.allow_install=true` → deny + non-zero exit |
| A3 | Allow with policy | With allow flag + `--confirm-schedule` → gate allow + audit row |
| A4 | Preview before install | Preview artifact written; diff includes scheduler command string |
| A5 | OS install (platform) | One platform supported (maintainer primary OS); idempotent re-run |
| A6 | Scheduler command | Installed entry invokes `run daily --unattended --safe` with config dir |
| A7 | Dry-run unchanged | `schedule install --dry-run` still no OS write; 0.4-d regression |
| A8 | Audit JSON | Gate decision file or SQLite row; schema fields per spec §5.3 |
| A9 | Unit tests | `pytest tests/unit/test_safety_gate.py -q` (or equivalent) green |
| A10 | Integration tests | `pytest tests/integration/test_schedule_install.py -q` green |
| A11 | No auto-approve | Gate never emits approve for pending review runs |

### Commands

```bash
loop-pilot schedule install --dry-run          # must still pass (0.4-d)
loop-pilot schedule install --yes --confirm-schedule   # 0.5-a (with allow_install)
loop-pilot safety doctor                       # optional: policy summary
python scripts/verify_0_5a_acceptance.py       # TBD skeleton
```

## 0.5-b — Unattended Safe Daily

### Scope

| In scope | Out of scope |
|----------|--------------|
| `run daily --unattended --safe` | `run daily` without `--safe` |
| Non-interactive pipeline | Real adapters in safe profile |
| Daily lock acquire/release | Background daemon |
| `daily_run.jsonl` log | Auto-commit / push |

### Checklist

| # | Item | Pass criteria |
|---|------|---------------|
| B1 | Flag pairing | `--unattended` without `--safe` → deny |
| B2 | Safe pipeline order | verify → recovery-scan → summary refresh logged in order |
| B3 | No adapter calls | Safe profile: no ToolBroker adapter invoke; test asserts |
| B4 | No real loops default | Default safe profile: no new intern/paper/daily-news run dirs |
| B5 | Lock semantics | Concurrent second invocation fails or waits per policy; no corrupt state |
| B6 | Summary updated | `daily-summary.md` refreshed; footer includes run metadata |
| B7 | Exit codes | Success 0; policy deny non-zero; failures auditable |
| B8 | Scheduler smoke | Manual or CI: invoke same command string as A6 |
| B9 | Integration tests | `pytest tests/integration/test_daily_unattended_safe.py -q` green |
| B10 | 0.4-d dry-run regression | `run daily --dry-run` unchanged |

### Commands

```bash
loop-pilot run daily --dry-run                 # 0.4-d regression
loop-pilot run daily --unattended --safe       # 0.5-b
python scripts/verify_0_5b_acceptance.py       # TBD skeleton
```
