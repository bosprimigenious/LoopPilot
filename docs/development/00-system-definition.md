# 00 系统定义

## 1. 系统目标

LoopPilot 是只服务项目所有者本人的本地 AI 工作闭环系统。它每天自动运行有限时间，处理三个实际问题：

1. 上班途中尝试解决昨天留下的一个真实开发问题。
2. 每天推动论文中最重要的一个研究、实验、引用或写作缺口。
3. 从 GitHub、研究信息和重大政治财经事件中提炼少量真正值得注意的内容。

## 2. 三条循环

| Loop | 地位 | 最终结果 |
|---|---|---|
| InternLoop | 主 Loop | 一个经过测试的代码修复或明确的阻塞诊断 |
| PaperLoop | 主 Loop | 一个有证据的论文推进结果或明确研究方向 |
| DailyNewsLoop | 小 Loop | 少量高价值信息与两个主 Loop 的候选任务 |

DailyNewsLoop 只能提供信息，不得直接修改代码或论文。

## 3. 必须具备的系统性质

- **真实**：必须读取真实仓库、真实测试、真实论文、真实实验和真实来源。
- **可控**：动作受路径、命令、时间、轮次、费用和人工审批约束。
- **闭环**：不是执行一次，而是验证失败后依据新证据有限重试。
- **可恢复**：中断后从检查点恢复。
- **可审计**：每个结论能追溯到日志、diff、实验或外部来源。
- **少而重要**：每次只处理一个主任务；日报不追求信息数量。
- **能停止**：证据不足、范围过大或风险过高时必须安全停止。

## 4. 完成的严格定义

### InternLoop 完成

代码已修改，任务验收条件满足，强制测试通过，没有越界 diff，并生成开发报告。仅生成补丁但未验证不算完成。

### PaperLoop 完成

选定缺口已修订，引用与论点可核验，实验数字一致，相关评价项通过，并生成论文开发报告。仅增加文字不算完成。

### DailyNewsLoop 完成

信息完成增量抓取、去重、日期核验、来源分级和重要性筛选，并生成不超过配置上限的日报。抓取大量链接不算完成。

## 5. 明确边界

- 不实现商业、团队、多租户和社交功能。
- 不自动 push、merge、部署、投稿或公开发布。
- 不自动改写原始实验结果。
- 不编造引用、数字、新闻事实和“首次”结论。
- 不把模型评分直接当作完成证据。
- 不把 Agent 数量当作系统能力。

## 6. 用户可见产出

所有用户需要阅读、判断和保存的内容均为 Markdown：

- 开发计划；
- 每轮诊断；
- 开发报告；
- 论文修订报告；
- 引用与证据报告；
- DailyNews 日报；
- 阻塞和审批报告；
- 每日总览。

内部 SQLite、JSONL 和 YAML 仅用于可靠运行，不替代 Markdown 结果。
