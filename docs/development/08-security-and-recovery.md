# 08 安全、审批与恢复

本分册描述最终安全目标。Mini 命中需要审批的规则时直接安全结束为 `BLOCKED`；V1 才实现持久化审批、`resume` 和事务恢复。

## 1. 默认策略

- 默认只读，执行阶段才获得最小本地写权限。
- 外部写入默认禁止。
- 无法判断安全性时停止。
- 密钥只由环境或系统凭据存储注入，不进入配置、Prompt、日志和报告。

## 2. 必须审批

- 修改计划外路径；
- 删除、移动或批量重命名；
- 新增或升级依赖；
- 修改数据库 migration、CI、部署和生产配置；
- diff 超过文件数或行数阈值；
- push、merge、发消息或其他远程写入；
- 改变论文核心结论、实验设定或数据解释；
- 证据矛盾或任务存在多种重大解释。

## 3. Policy Decision

每次决策输出：

```text
ALLOW / REQUIRE_APPROVAL / DENY
rule_id
target
reason
evidence
approved_scope
expires_at
```

审批只对当前 Run、当前动作和明确范围有效。

## 4. 故障处理

| 错误 | 行为 |
|---|---|
| 临时网络/模型错误 | 指数退避、有限重试 |
| 配置/Schema/权限错误 | 立即停止 |
| 工具超时 | 终止子进程，保留部分输出 |
| 磁盘/数据库错误 | 停止写入并保护现有状态 |
| 工作区外部变化 | 校验哈希后请求确认 |
| 报告失败 | 保留任务结果，单独重建报告 |

## 5. 恢复

1. 校验 Run、检查点和工作区哈希。
2. 确认上一个事务已提交。
3. 检查外部副作用是否发生。
4. 创建新 `attempt_id`。
5. 从最后一个合法状态继续。
6. 在最终报告中记录中断和恢复过程。

## 6. 安全停止报告

必须说明停在哪、已做什么、未做什么、工作区是否安全、恢复条件和推荐命令。任何失败都不能留下“可能改了一半但不知道”的状态。
