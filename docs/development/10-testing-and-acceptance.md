# 10 测试与验收

## 1. 测试层次

- 单元：状态、预算、锁、策略、Schema、排序、去重、模板。
- 集成：CLI 到存储、模型、Git worktree、论文副本、Connector。
- 场景：三个 Loop 的真实成功与阻塞路径。
- 故障注入：超时、断网、磁盘、数据库、子进程、外部修改。
- 回归：固定样例仓库、论文和新闻数据集。

## 2. InternLoop 强制场景

- 修复一个失败测试。
- 第二轮依据日志修复成功。
- 任务过大时缩小或停止。
- 命中危险路径时请求审批。
- 用户存在未提交改动时不覆盖。
- 依赖缺失时不擅自安装。
- 测试通过但 diff 越界时不宣布成功。

## 3. PaperLoop 强制场景

- 定位并修复 unsupported claim。
- 缺引用时查询并核验候选来源。
- 无法核验时写 SOURCE REQUIRED 并停止。
- 发现正文与实验数字不一致。
- 给出缺失实验而不伪造结果。
- 修订后重新评价并显示诊断变化。
- 证据冲突时请求用户判断。

## 4. DailyNewsLoop 强制场景

- 多来源同事件去重。
- 区分事件发生日期和文章发布时间。
- 社区消息追溯原始来源。
- 低置信度消息不进入正式 Top News。
- 第一次 GitHub 快照不声称“24 小时增长第一”。
- 第二次快照正确计算 Star 增量。
- 无重大新闻时输出空类别而不是凑数。

## 5. 故障注入

- 模型超时、限流和结构化输出错误；
- API/RSS/网页不可达；
- Mini JSON 状态写失败；
- V1 SQLite 事务写失败；
- 磁盘不足；
- 测试进程挂起；
- 工作区被外部修改；
- 报告渲染中断；
- 锁遗留；
- 非法状态恢复。

## 6. 总验收矩阵

| 能力 | 证据 |
|---|---|
| 30 分钟硬停止 | 计时场景与终态报告 |
| 三轮有限重试 | 事件日志与 Round 报告 |
| 真实代码修复 | patch、测试日志、diff review |
| 真实论文推进 | 修订 diff、引用和评价报告 |
| 高价值日报 | 来源、去重、排名和条目上限 |
| 风险可控 | Policy 测试与审批记录 |
| 可恢复 | 中断恢复场景 |
| 可审计 | 完整 Artifact 链 |

## 7. 发布到日常运行链的门槛

- 所有单元和强制场景通过。
- 没有未解释的高严重度故障。
- 成功、阻塞、失败报告均可读且事实正确。
- 密钥和敏感信息扫描通过。
- 连续多次运行没有重复任务、锁死和状态污染。
