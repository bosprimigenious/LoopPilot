# 0.5.2 Source Policy and Evaluator Bridge Alignment

Date: 2026-06-23

Branch: `main`

## Scope

This log records the planning and documentation update that aligns LoopPilot's
0.5.2 line with `agentic-rubric-runner`'s fine-grained version governance and
evaluation-gate direction.

The key decision is to keep these items in the 0.5.2 line rather than pushing
them into 0.6:

```text
0.5.2.4.x  Web Evidence / Source Policy Alignment
0.5.2.5.x  LoopPilot Evaluation Bridge
0.6.0      Real write-capable agents after gates are closed
```

## Decisions

### 1. Version Governance

LoopPilot now follows the same fine-grained planning style as
`agentic-rubric-runner`:

```text
Public release:  0.5.2
Internal slice:  0.5.2.1
Gate iteration:  0.5.2.1.3
Evidence run:    0.5.2.1.3.2
```

`0.5-a / 0.5-b / 0.5-c / 0.5-d` are legacy labels only and must not be used to
declare current 0.5 readiness.

### 2. Source Policy Belongs to 0.5.2.4.x

LoopPilot may discover, fetch, read, and normalize sources, but it must not
claim that source-derived content is trustworthy without evaluator review or
human review.

The source policy fields are:

```yaml
mode: replay | live
live: false | true
source_use: discovery_only | evidence_allowed | report_allowed
evaluator_required: true | false
```

Replay fixtures that use source kinds such as `github`, `arxiv`, or `rss` must
still declare `mode: replay` and `live: false`.

### 3. Evaluation Bridge Belongs to 0.5.2.5.x

The LoopPilot-to-evaluator bridge is a separate slice from source policy.

The planned bridge artifacts are:

```text
evaluation_request.json
evaluation_result.json
```

LoopPilot should eventually emit `evaluation_request.json`, read
`evaluation_result.json`, and map evaluator output to:

```text
pass
needs_review
blocked
```

When evaluator output is required but missing or invalid, LoopPilot acceptance
must fail closed.

backward-compatible, but this should still be covered by the next full gate run.
