### config.local 用法（推荐）

LoopPilot 默认读取 `config/`。本机 live run **不要**把 key 写进仓库 YAML。

**方式 A — 仅 CLI opt-in（推荐，无需改 config）：**

```bash
loop-pilot run paper --workspace examples/paper_demo --adapter deepseek --allow-real-adapters --dry-run
loop-pilot run intern --workspace examples/intern_demo --adapter cursor_cli --allow-real-adapters --dry-run
```

**方式 B — 本机 config 目录（不提交）：**

1. 复制 `config/` 到仓库外或 `config.local/`（加入 `.gitignore`）
2. 在副本的 `loop-pilot.yaml` 中可选设置 `runtime.allow_real_adapters: true`（仍建议配合 CLI flag）
3. 运行时使用 `--config-dir /path/to/config.local`

示例 `config.local/loop-pilot.yaml` 片段（**无 API key 字段**）：

```yaml
runtime:
  allow_real_adapters: false   # 保持 false；live run 用 CLI --allow-real-adapters
  artifact_dir: var/artifacts
tool_broker:
  command_timeout_seconds: 300
  allowed_commands: [pytest, python, git]
```

Key 仅通过 `auth_env` 对应的环境变量注入（见 `config/adapters.yaml` 中 `deepseek.auth_env: DEEPSEEK_API_KEY`）。
