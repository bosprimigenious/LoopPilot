## 5. 论文 / benchmark 风险

### 5.1 SWE-Bench Mobile

- 链接：[arXiv:2602.09540 — SWE-Bench Mobile](https://arxiv.org/abs/2602.09540)
- 重点：
  - 工业级 mobile app 开发 benchmark。
  - 搜索摘要显示最好配置 task success rate 仍很低。
- 对 LoopPilot：
  - 支持“真实复杂工程需要 runtime control、audit 和 human review gate”的论点。

### 5.2 SWE-Skills-Bench

- 链接：[arXiv:2603.15401 — SWE-Skills-Bench](https://arxiv.org/abs/2603.15401)
- 重点：
  - 大量 skills 并没有带来 pass-rate 改善。
  - Skill 注入可能增加 token overhead 或造成版本冲突。
- 对 LoopPilot：
  - 不应盲目堆 skill；必须用 evaluation gate 判断 skill 是否真的有效。

### 5.3 MarketBench

- 链接：[arXiv:2604.23897 — MarketBench](https://arxiv.org/abs/2604.23897)
- 重点：
  - agent 对成功概率与成本自评不准。
- 对 LoopPilot：
  - 支持“agent 不得自证 READY”的设计。

### 5.4 AIDev dataset

- 链接：[arXiv:2602.09185 — AIDev: Studying AI Coding Agents on GitHub](https://arxiv.org/abs/2602.09185)
- 重点：
  - 大规模真实 agent-authored PR 数据。
- 对 LoopPilot：
  - 可作为 evaluation 层和 InternLoop 风险模型的数据来源。

### 5.5 AI Bot CI/CD reliability

- 链接：[arXiv:2604.18334 — Reliability of AI Bots Footprints in GitHub Actions CI/CD Workflows](https://arxiv.org/abs/2604.18334)
- 重点：
  - agent PR 在 CI/CD 中可靠性不同。
- 对 LoopPilot：
  - 可支持 ToolBroker、CI gate、review gate 的必要性。

## 9. 本轮 DailyNews 对 LoopPilot 的直接结论

当前项目里的 DailyNewsLoop 还不能生产这份真实日报，因为它缺少：

- live GitHub source；
- live RSS/news source；
- cross-source verification；
- unresolved signal artifact；
- politics / finance report sections；
- true 30-minute multi-round runner；
- source freshness and correction mechanism。

下一步最合理目标不是“立刻自动联网跑”，而是：

```text
DailyNews 0.5-live-small:
  local-first
  read-only network
  source whitelist
  no auto Inbox import
  unresolved by default when confidence is weak
  explicit human review before routing to Intern/Paper
```
