# 2026-06-27 0.5.4 Three-Loop Safe Auto Runner Plan

## Context

The supervisor feedback clarified that the 0.5 line must not only define gates,
schemas, and bridge contracts. It must also make the three core loops actually
run automatically enough for personal daily use.

The important boundary is:

```text
0.5.x = safe personal automatic loop runs
0.6.x = real write-capable external agents
```

## Decision

Place the automatic three-loop runner in `0.5.4.x`.

Do not move it to `0.6.0` or `0.7.0`.

Rationale:

- `0.5.2.x` is already responsible for runtime contracts, source policy, and the
  evaluator bridge.
- `0.5.3.x` is the right place for read-only connector prototypes.
- `0.5.4.x` is the first version where the project has enough gates and inputs
  to run DailyNewsLoop, InternLoop, and PaperLoop automatically in safe mode.
- `0.6.x` should stay reserved for real external write actions: branch creation,
  code edits, commits, pushes, draft PRs, paper edits, and trusted live news
  publication.

## Version Breakdown

```text
0.5.2.x
Runtime artifact contracts, source policy, and evaluation bridge.

0.5.3.x
Read-only connector prototypes for GitHub, Paper/Data, and DailyNews.

0.5.4.x
Safe personal automatic runner for all three core loops.

0.6.x
Real write-capable agents.
```

Detailed `0.5.4.x` slices:

```text
0.5.4.1 run-all personal safe orchestration
0.5.4.2 unattended daily safe entry
0.5.4.3 loop budgets, locks, retry limits, and partial success semantics
0.5.4.4 aggregate evaluator gate
0.5.4.5 five-day dogfood evidence
```

## Required User-Facing Command

```bash
loop-pilot run daily --unattended --safe --profile personal
```

This command must run:

```text
DailyNewsLoop -> InternLoop -> PaperLoop
```

## Gate Requirements

`0.5.4` can be called READY only when:

- all three loops run from one command;
- every loop emits terminal artifacts;
- every loop has an artifact manifest, gate result, and terminal reason;
- evaluator-required profiles fail closed when `evaluation_result.json` is
  missing or invalid;
- unsafe external mutations are blocked;
- the aggregate result is one of `pass`, `needs_review`, or `blocked`;
- five consecutive safe daily dogfood runs are documented.

## Explicit Non-Goals

The following remain 0.6:

- creating branches;
- editing code;
- committing or pushing;
- opening draft PRs;
- editing papers or figures;
- publishing verified live news reports;
- importing unverified live findings without review.
