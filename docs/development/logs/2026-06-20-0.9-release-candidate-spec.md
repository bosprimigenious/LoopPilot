# 规划日志：0.9 Release Candidate / Stability Freeze 规格确立

**日期**：2026-06-20  
**决策类型**：版本边界、稳定化工程与 RC 流程  
**状态**：已采纳（文档 only，未实现代码）

## 背景

0.5–0.8 路线图扩展后，LoopPilot 在 **0.8 team-cloud-preview** 时功能面已接近完整（预估 **≈95%+**），但缺少一个**专门的稳定化阶段**：

- 0.5 聚焦「别人能安装」；0.6–0.8 聚焦「扩展与协作 preview」
- 若在 0.8 后直接发 1.0，API/配置/DB 仍可能变动，安全与长跑可靠性未系统验收
- 旧 V3「长期稳定」被简单映射到 1.0，未区分「最后验收（0.9）」与「正式承诺（1.0）」

## 决策

### 1. 新增 0.9 release-candidate

| 版本 | 定义 |
|------|------|
| **0.8** | 团队/项目协作 preview（非完整 SaaS） |
| **0.9** | **最后验收**：冻结、测试、修复、兼容、清理、文档定稿 |
| **1.0** | **正式 stable 承诺**（ideal：自末位 rc 零功能 diff） |

**0.9 核心不是加功能**，是稳定化工程。

### 2. 版本命名

- Git tag / GitHub Release：`0.9.0-release-candidate` 或 `0.9.0-stability-freeze`
- PyPI 预发布：`0.9.0rc1` / `0.9.0rc2` / `0.9.0rc3`（PEP 440 RC）
- RC 晋级：rc(N+1) 仅当 rc(N) 无 open P0 且 conformance CI 连续 7 天绿

完整链：

```text
0.1.0-mini → … → 0.8.0-team-cloud-preview
  → 0.9.0-release-candidate → 1.0.0-stable
```

### 3. 0.9 必须实现（写入 34 §10）

1. **API Freeze** — `docs/stability/api-freeze.md`
2. **Configuration Freeze** — `schema_version`、`config validate/migrate/doctor`
3. **DB Migration Freeze** — `db/migrations/`，0.9 起仅 additive
4. **Conformance Test Suite** — `conformance/` + CI 门禁
5. **Security Audit** — `SECURITY_REVIEW.md` / `docs/stability/security-review-0.9.md`
6. **Performance / Reliability** — 7 天模拟长跑 + 故障注入
7. **Documentation Freeze** — 用户文档与 CLI 一致
8. **Deprecation Policy** — `docs/stability/deprecation-policy.md`

### 4. 新增模块规划（目录级，0.9 实现时填入）

| 目录 | 职责 |
|------|------|
| `src/loop_pilot/stability/` | compat 检查、RC 报告 |
| `config/schema_versions/` | schema 快照与迁移 |
| `src/loop_pilot/db/migrations/` | SQLite migration |
| `conformance/` | 契约测试 |
| `docs/stability/` | freeze / deprecation / security |
| `security/` | 审计脚本、CVE 扫描 |

### 5. 0.9 明确不做

新 Loop、新 Web UI、云端、插件市场、企业 SSO、破坏性变更、新 Adapter 类型（仅 bugfix）。

### 6. 0.9 vs 1.0

| | 0.9 | 1.0 |
|---|-----|-----|
| 性质 | 最后验收；rc 标签 | 正式 stable；semver 承诺 |
| 变更 | bugfix、文档、兼容 | patch/security |
| 破坏性 | 禁止（→ 2.0） | 禁止（→ 2.0） |

### 7. 0.6 / 0.7 / 0.8 简要定位（同批确立）

| 版本 | 名称 | 核心 |
|------|------|------|
| 0.6 | plugin-ecosystem | 本地插件、自定义 Loop/Skill/Connector |
| 0.7 | evaluation-benchmark | benchmark、模型对比、Eval report |
| 0.8 | team-cloud-preview | 项目工作区、角色权限、Dashboard |

### 8. Legacy 映射更新

| 旧称呼 | 新编号 |
|--------|--------|
| V2 插件 | **0.6** |
| V2 benchmark / 模型对比 | **0.7** |
| V2/V3 团队/协作 preview | **0.8** |
| V3 稳定化 / freeze / 安全 | **0.9** |
| V3 正式长期承诺 | **1.0** |

### 9. 完成度估计

- 到 **0.8**：≈**95%+**（自项目起点）
- 到 **0.9**：自 0.8 完成点还差 **15–25%**（稳定化工程，约 2–4 周）
- 到 **1.0**：0.9 rc3（或末位 rc）全部门禁通过
