# LoopPilot Truthful 0.4 验收报告

**日期：** 2026-06-21  
**验收层级：** 第一层 — Truthful 0.4 Acceptance  
**最终结论：** **NOT READY**  
**判定原因：** 0.4-c Review Layer 尚未实现，验收结果为 `0/16`。0.4-d 已具备自身功能，但不得越过失败的前置版本宣布 READY。

## 1. 验收目标

本轮重点验证 LoopPilot 是否满足以下可信性要求：

1. 前置验收失败时，后续版本不会误报 READY。
2. 静态文件齐全但运行时检查失败时，不会误报 READY。
3. 验收脚本以所有必需检查的真实结果决定状态和退出码。
4. 修复不会破坏现有测试、静态检查或代码格式。

本轮不宣称完成 Ideas Acceptance。Intern、Paper、DailyNews 的真实任务闭环仍需单独验收。

## 2. 发现的 P0 缺陷

### P0：0.4-d 可绕过 0.4-c 失败并误报 READY

修复前的实际结果：

```text
0.4-c acceptance (NOT READY): 0/16 passed
0.4-d acceptance (READY): 29/29 passed
0.4-d exit code: 0
```

这意味着 0.4-c 完全未交付时，0.4-d 仍可宣布 READY。该行为违反 Truthful Acceptance 的核心规则，也会使 CI 或人工验收错误地接受不完整版本。

### 根因

1. `verify_0_4d_acceptance.py` 没有执行 0.3、0.4-b、0.4-c 前置验收。
2. 0.4-b、0.4-c、0.4-d 的 READY 状态只依赖部分静态 readiness 项。
3. pytest、数据库验证、CLI smoke 等运行时步骤即使失败，标题仍可能显示 READY；仅退出码变为失败。

## 3. 已实施修复

### 3.1 强制前置验收链

0.4-d 现在依次执行：

```text
verify_0_3_acceptance.py
verify_0_4b_acceptance.py
verify_0_4c_acceptance.py
```

任一前置验收失败时：

- 0.4-d 标记为 NOT READY；
- 返回非零退出码；
- 停止执行后续 0.4-d 验收步骤；
- 输出具体失败的前置版本，不用模糊的总失败掩盖原因。

### 3.2 READY 判定收紧

0.4-b、0.4-c、0.4-d 现在都要求：

```python
bool(results) and all(result.passed for result in results)
```

只要任何已记录检查失败，就不得打印 READY。

### 3.3 反作弊回归测试

新增 `tests/unit/test_acceptance_truthfulness.py`，覆盖：

- 0.4-b 运行时失败不得 READY；
- 0.4-c 运行时失败不得 READY；
- 0.4-d 运行时失败不得 READY；
- NOT READY 必须返回非零退出码；
- 0.4-c 前置失败时，0.4-d 必须停止；
- 0.4-d 确实执行 0.4-c 验收脚本。

专项测试结果：

```text
7 passed in 0.01s
```

## 6. 尚未满足的门禁

Truthful 0.4 目前仍不能宣告 READY，至少需要：

1. 实现并通过 0.4-c Review Layer 的 16 项验收。
2. 补齐 SQLite v1/v2/v3/v4 migration matrix。
3. 补齐全失败路径 Artifact 完整性矩阵。
4. 补齐真实 kill/crash/recovery/resume 测试。
5. 补齐并发运行、跨进程锁与 run ID 冲突测试。
6. 验证人工 Review 决策不可绕过。

Ideas Acceptance 还需另行完成：

- InternLoop 真实 Git repo、失败测试、修复、pytest、git diff 闭环；
- PaperLoop Claim–Evidence–Citation 正反例；
- DailyNews recorded replay 与可选 live connector；
- 预算、超时、重试、取消；
- 连续 7 天 dogfooding。

## 7. 最终结论

本轮没有证明 LoopPilot 0.4 已完成；本轮证明的是验收系统开始能够诚实地拒绝不完整版本。

当前准确表述应为：

> LoopPilot 的现有测试与 0.4-b 验收通过，0.4-d 自身功能可运行；但 0.4-c Review Layer 尚未实现，因此 Truthful 0.4 总验收为 **NOT READY**。0.4-d 的跨版本 READY 误报已修复，并由反作弊测试覆盖。
