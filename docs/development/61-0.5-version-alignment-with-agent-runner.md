# 0.5 Fine-Grained Version Plan Aligned with agentic-rubric-runner

本文件把 LoopPilot 的 0.5 迭代方式改成和 `agentic-rubric-runner`
一致的“短公开版本 + 长内部版本”体系。

This document aligns LoopPilot 0.5 planning with the version governance style
used by `agentic-rubric-runner`: short public releases, long internal iteration
versions.

当前 LoopPilot 基线：

当前 agentic-rubric-runner 公开基线：

不要继续把新的公开规划写成 `0.5-a / 0.5-b / 0.5-c / 0.5-d`。这些只保留为历史子阶段标签。

## Version Layers / 版本层级

| Layer | Format | Example | Meaning |
|---|---|---|---|
| Public release / 公开发布 | `MAJOR.MINOR.PATCH` | `0.5.2` | README badge, GitHub release, package-facing version |
| Internal slice / 内部切片 | `MAJOR.MINOR.PATCH.SLICE` | `0.5.2.1` | One implementation slice under a public patch |
| Gate iteration / 门控迭代 | `MAJOR.MINOR.PATCH.SLICE.GATE` | `0.5.2.1.3` | Acceptance, manifest, baseline, policy, or evaluator-gate refinement |
| Evidence run / 证据运行 | `MAJOR.MINOR.PATCH.SLICE.GATE.RUN` | `0.5.2.1.3.2` | A documented verification, benchmark, dogfood, or acceptance run |

Use long versions for planning, iteration logs, handoff prompts, and gate
evidence. Use short versions for public installable releases unless packaging
explicitly requires a longer identifier.

长版本用于计划、迭代日志、交接提示和门控证据；正式可安装发布默认使用短版本，除非打包体系明确需要长标识。

```text
| +---------- minor line: safe runtime + evaluation-gated agentic work / 0.5 主线
+------------ major line: 0.x personal-first stage / 0.x 个人优先阶段
```

## Alignment Rule / 对齐规则

LoopPilot is the runtime layer. `agentic-rubric-runner` is the evaluation layer.

LoopPilot 是运行层；`agentic-rubric-runner` 是评价层。

The two projects do not need identical code releases, but their planning lines
should be readable together:

两边不需要同步发完全相同的包版本，但规划线要能读成一组：

```text
agentic-rubric-runner 0.5.2 = truthful benchmark gates and evaluator release gate
LoopPilot 0.5.2            = runtime artifacts and gate contracts that can be judged by that evaluator style
```

In other words:

换句话说：

```text
agentic-rubric-runner is responsible for "judge".
LoopPilot is responsible for "run".
The shared version story is about making every run evaluable, auditable, and regression-testable.
```

```text
agentic-rubric-runner 负责“判”。
LoopPilot 负责“跑”。
共同版本故事是：每次运行都必须可验收、可审计、可回归。
```

## READY / NOT READY Rules

Allowed:

允许：

```text
0.5.2.1.3 implemented; 0.5.2 aggregate NOT READY
0.5.2.2 artifact contract PASS; real agents still BLOCKED until 0.6
0.5.2.1.3.2 evidence run: NOT READY due to missing live evaluator
```

Forbidden:

禁止：

```text

0.5.2 READY
real agents connected
unattended production ready
```

unless the aggregate gate and all required evidence runs pass.

除非聚合门控和所有必需证据运行都通过，否则不得这样写。
