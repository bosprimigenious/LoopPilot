## 0.4 差距清单

### 0.4-G1：Personal profile 不完整

问题：

- 默认配置仍是 JSON；
- sqlite 需要用测试 fixture config 或手动改配置；
- 没有清晰的个人日用 profile。

应补：

```text
config/profiles/personal-sqlite/
config/profiles/smoke-sqlite/
```

或者：

```bash
loop-pilot init personal
loop-pilot doctor --profile personal
```

验收：

```bash
loop-pilot --config-dir config/profiles/personal-sqlite doctor
loop-pilot --config-dir config/profiles/personal-sqlite db status
```

### 0.4-G2：runtime path 隔离不完整

问题：

- `--config-dir` 不等于状态隔离；
- 相对 `state_dir` / `artifact_dir` / `sqlite_path` 仍可能落到仓库 `var/`。

应补：

```yaml
runtime:
  root_dir: ~/.loop-pilot/personal
  state_dir: state
  artifact_dir: artifacts
  sqlite_path: state/loop_pilot.db
```

并统一路径解析：

```text
resolve_runtime_path(config_dir, runtime.root_dir, path)
```

验收：

- 临时 config 不读取仓库 `var/`；
- 不同 profile 的 run 互不污染；
- `db status` 显示绝对路径。

### 0.4-G3：Today CLI 不完整

问题：

`today` 现在更像写入口，而不是每日工作面板。

应补：

```bash
loop-pilot today list
loop-pilot today show <item_id>
loop-pilot today done <item_id>
loop-pilot today defer <item_id> --until YYYY-MM-DD
```

验收：

- 空 today 输出清楚；
- `today list` 与 `summary today` 一致；
- done/defer 后 summary 同步变化。

### 0.4-G4：Review UX 需要收紧

问题：

- approve 目前 direct-finalize；
- resume 文案仍容易让人误以为 approve 后还要 resume。

应补：

- CLI help；
- docs；
- `review show` next-action hint。

推荐语义：

```text
approve = finalizes reviewed patch run
resume = only for interrupted/recoverable runs
```

### 0.4-G5：失败报告可读性不足

问题：

- blocked report 太薄；
- 人读 report 不能快速知道下一步。

应补：

```markdown
## Command
## Outcome
## Gate
## Reason
## External writes
## Artifacts
## Next action
```

### 0.4-G6：DailyNews 仍是离线筛选器

问题：

- `live-small` 仍是 local JSON；
- 没有真实 GitHub/arXiv/RSS；
- 没有聚簇；
- 没有 cross-source verification。

归属：

- 0.5.3 做只读 connector foundation；
- 0.6.2 做真实 DailyNews。

### 0.4-G7：Intern/Paper 仍是 fixture/demo

问题：

- Intern 不读真实 PR/CI；
- Paper 不读服务器数据；
- 两者都不会开真实 draft PR。

归属：

- 0.5.2.4.x 做 source policy / replay-live 标记；
- 0.5.2.5.x 做 LoopPilot evaluation bridge；
- 0.5.3.x 再做只读 connector prototypes；
- 0.6.0 / 0.6.1 做真实写动作。

## 0.5 开发目标

`0.5 = Personal Runtime Substrate`

它不是 Real Agent，而是真实运行地基。

## 0.5 非目标

0.5 不做：

- 自动修真实 PR；
- 自动 push；
- 自动 open PR；
- 自动修改 paper；
- 自动联网成正式日报；
- 自动 approve；
- 自动 merge；
- 默认 OS schedule 真运行。

## 0.5 开发分解

### 0.5.2.3 Personal Runtime Substrate

必须完成：

- personal SQLite profile；
- smoke SQLite profile；
- runtime path isolation；
- secrets redaction；
- improved report templates；
- today read model；
- review UX polish；
- schedule still fail-closed；
- branch/PR/source contract docs。

建议代码模块：

```text

src/loop_pilot/runtime/paths.py
src/loop_pilot/tasks/today_service.py
src/loop_pilot/cli_tasks.py
src/loop_pilot/reporting/
src/loop_pilot/runtime/terminal_artifacts.py
```

验收脚本：

```text
scripts/verify_0_5_2_3_acceptance.py
```

验收命令：

```bash

pytest -q

```

### 0.5.3.1 GitHub Read Connector Prototype

目标：

只读 GitHub，不创建分支、不 push、不开 PR。

必须能力：

- `gh auth status`；
- read repo metadata；
- read PR list；
- read issue list；
- read CI check status；
- read PR review comments；
- save raw response as redacted artifacts。

建议命令：

```bash
loop-pilot github doctor
loop-pilot github repo status --repo owner/name
loop-pilot github pr list --repo owner/name
loop-pilot github pr inspect <number>
loop-pilot github ci inspect <number>
```

必须限制：

```text
No branch creation.
No commit.
No push.
No PR creation.
```

验收：

- missing auth -> BLOCKED；
- dry-run writes artifacts only；
- secrets redacted；
- no network write calls。

### 0.5.3.2 Paper/Data Read Connector Prototype

目标：

只读 paper repo / data manifest / server path，不改论文。

必须能力：

- read paper repo status；
- read manuscript files；
- read claim list；
- read experiment plan；
- read data manifest；
- compute checksums；
- detect new data；
- write `paper-data-scan-report.md`。

建议命令：

```bash
loop-pilot paper scan --repo <path-or-url> --claim-list claims.yaml --data-manifest data/manifest.json
```

必须限制：

```text
No paper edit.
No figure update.
No branch.
No commit.
No push.
```

### 0.5.3.3 News Read Connector Prototype

目标：

只读真实/半真实 news source，生成 replay artifacts，不生成最终可信日报。

必须能力：

- GitHub read source；
- arXiv read source；
- RSS read source；
- source whitelist；
- source tier；
- raw source snapshot；
- preliminary unresolved signals。

建议命令：

```bash
loop-pilot news doctor
loop-pilot news fetch --profile live-small --dry-run
loop-pilot run daily-news --source-profile live-small --dry-run
```

必须限制：

```text
No auto Inbox import.
No final factual claim without verification.
No C-tier single-source final report.
```

## 0.5 Acceptance Matrix

| Gate | 目标 | 必须结果 |
|---|---|---|
| 0.4 regression | 0.4 不退化 | `verify_0_4_acceptance.py` READY |
| path isolation | profile 不串状态 | temp config 不读 repo `var/` |
| personal sqlite | 日用 profile 可用 | doctor/db/status OK |
| today UX | 今日任务可见 | list/show/done/defer OK |
| report quality | 失败可读 | blocked report 有 reason/next action |
| schedule safety | fail-closed | prep 阶段 install/unattended blocked |
| GitHub read | 只读接入 | no branch/no push/no PR |
| Paper data read | 只读扫描 | no edit/no branch/no push |
| News read | 只读 source | raw snapshot + unresolved |
| secrets | 不泄漏 | token redacted |

## 0.5 完成定义

0.5 完成时，可以说：

```text
LoopPilot can safely connect to real-world read surfaces.
It has isolated personal runtime state, profile-aware paths, redacted credentials,
clear reports, and fail-closed scheduling.
```

不能说：

```text
LoopPilot can autonomously fix GitHub PRs.
LoopPilot can update real papers.
LoopPilot can produce verified live news reports.
```

这些必须留给 0.6。
