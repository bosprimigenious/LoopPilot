# 09 Mini、V1、V2、V3 开发范围

版本表示开发依赖和可验收能力，不表示商业发布。

## Mini：证明闭环真的成立

### 范围

- CLI 手动运行。
- 文件式简化状态和 Markdown 报告。
- 统一三轮、30 分钟控制。
- InternLoop 在测试样例仓库修一个预置 Bug。
- PaperLoop 修一个预置 unsupported claim 或缺引用段落。
- DailyNewsLoop 从少量固定源生成去重日报。

### 验收

- 三个 Loop 都能从 Observe 走到 Report。
- InternLoop 的修复通过真实测试。
- PaperLoop 不编造引用，并能在缺来源时停止。
- DailyNews 不重复输出同一条目。
- 失败也能生成可读报告。

## V1：可用于每天真实工作

### 范围

- SQLite 状态、锁、检查点和恢复。
- Git worktree 与论文工作副本。
- Policy Engine、命令白名单和 diff 检查。
- 昨日任务优先和两个 Inbox。
- 真实仓库测试链与论文评价链。
- OS 定时运行和每日总报告。

### 验收

- 能在真实仓库解决一个有限任务而不污染主工作区。
- 能推进真实论文的一个缺口并给出下一方向。
- 每次约 30 分钟硬停止且保留 5 分钟收尾。
- 中断后可恢复；重复触发不会重复写入。

## V2：扩大查询和验证能力

### 范围

- GitHub、论文元数据、正式文献与新闻 Connector。
- GitHub Star 每日快照和增量排名。
- 引用元数据核验、全文证据位置和文献矩阵。
- 政治财经原始来源与交叉核验。
- 多模型路由、费用统计和备用策略。
- 故障注入与固定回归场景集。

### 验收

- “今日热门 GitHub”基于真实快照增量，而非总 Star 猜测。
- 论文引用能追溯到稳定标识和支持位置。
- 重大新闻区分事实、推断、发生时间和发布时间。
- Connector 失败不会破坏主 Loop。

## V3：长期稳定与高质量决策

### 范围

- 更稳定的任务优先级学习，但用户规则始终最高。
- 跨日问题链、研究方向演化和历史诊断比较。
- InternLoop 更完整的回归、静态分析与风险评估。
- PaperLoop 的 Reviewer Simulation、实验设计建议和一致性追踪。
- 长期运行健康检查、缓存和保留策略。

### 验收

- 连续运行期间无未解释外部写入、状态丢失和重复任务。
- 每日报告能准确说明“最重要的一件事”，而不是罗列信息。
- 历史证据能解释为什么今天选择当前任务和研究方向。
- 三条 Loop 的成功、阻塞和失败语义长期一致。

## 版本推进规则

一个版本只有全部强制场景通过才可作为下一个版本的基础。允许提前开发独立组件，但不得提前接入每日主运行链。
